# Security Policy

## Supported Versions

The following versions of PulsarTrack are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of PulsarTrack seriously. If you believe you have found a security vulnerability, please report it to us privately.

**Please do not report security vulnerabilities via public GitHub issues.**

Instead, please send an email to security@pulsartrack.io. We will acknowledge receipt of your report within 48 hours and provide a follow-up timeline for triage and remediation.

### What to include:
- A description of the vulnerability.
- Steps to reproduce the issue (reproducible scripts/curl commands are highly appreciated).
- Potential impact analysis.

## Scope

The following components are in scope for security reports:
- PulsarTrack Smart Contracts (Soroban/Rust)
- Backend API (Express)
- Frontend Application (Next.js)

Out of scope:
- Third-party dependencies (managed via updates)
- The Stellar Network itself

## Safe Harbor

We promise not to take legal action against researchers who report vulnerabilities in good faith and follow this policy. We appreciate your help in keeping the PulsarTrack ecosystem safe!
